# Domain documentation

LifeLedger uses a single domain context.

Before significant engineering work, read:

1. `CONTEXT.md` for product vocabulary and domain boundaries.
2. Relevant files under `docs/adr/` for architecture decisions.

Use the terms defined in `CONTEXT.md` in issue titles, tests and implementation notes. If a decision conflicts with an ADR, explicitly surface the conflict instead of silently overriding it.
